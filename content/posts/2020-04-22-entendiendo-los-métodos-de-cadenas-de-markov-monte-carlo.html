Una de las cosas m√°s atractivas de la inferencia bayesiana es que permite la incorporaci√≥n de conocimiento previo o creencias mediante distribuciones de probabilidad sobre los par√°metros de inter√©s. Por ejemplo, uno querr√≠a estimar cu√°l es la tasa de letalidad de una enfermedad y querr√≠a incorporar la informaci√≥n de un estudio previo, que indica que es del 0.5%. Esto confronta con la escuela frecuentista de la estad√≠stica, que asume que las probabilidades est√°n estrictamente relacionadas con la frecuencia de los eventos. Para los frecuentistas, el valor real de este par√°metro (por ej, la tasa de letalidad) es fijo y desconocido y, por tanto, no tiene sentido establecer ninguna distribuci√≥n de probabilidad sobre √©l.

Esto se traduce en que, por ejemplo, los intervalos de confianza de la estad√≠stica frecuentista tengan una interpretaci√≥n que a muchos no les parece tan natural. Por ejemplo, para el caso de la letalidad, dir√≠amos que nuestro intervalo al 90% de confianza contendr√° en un 90% de las veces al valor real de la tasa de letalidad. En otras palabras, si tomamos 100 muestras independientes y calculamos el intervalo de confianza para cada una de ellas, la esperanza es que 90 de cada 100 contendr√≠an el valor real. Esta es una excelente visualizaci√≥n que muestra lo anterior.

Para muchos, sin embargo, es m√°s natural preguntarse: ¬øcu√°l es la probabilidad de que mi par√°metro est√© dentro de un cierto intervalo? Esto es posible bajo el prima bayesiano. En la concepci√≥n bayesiana de la estad√≠stica se acepta que, dado que nuestro conocimiento sobre el valor real del par√°metro en cuesti√≥n es limitado, tiene sentido establecer distribuciones de probabilidad sobre √©l. La tasa de letalidad puede ser del 0.4% con una probabilidad \(P_1\), 0.5% con una probabilidd \(P_2\), etc.

Inferencia bayesiana: problemas y soluciones

La estad√≠stica bayesiana, se basa en el Teorema de Bayes:

\[ P(\theta | x) = \frac{P(x|\theta)P(\theta)}{P(x)} \]

\(P(\theta | x) = \frac{P(x|\theta)P(\theta)}{\int_{\theta}{}P(x,\theta)d\theta}\)

En donde \(P(\theta | x)\) es la probabilidad a posteriori del par√°metro dados los datos \(x\) (esto es lo que queremos estimar), \(P(x|\theta)\) es la funci√≥n de verosimilitud e indica la probabilidad de observar \(x\) dado \(\theta\) y \(P(\theta)\) refleja la creencia a priori sobre nuestro par√°metro \(\theta\). El denominador, que en la versi√≥n discreta ser√≠a \(P(x) = \sum_{\theta}P(x,\theta)\) (se le suele denominar la evidencia), es precisamente el elemento que complica el c√°lculo de la ecuaci√≥n. La integral que se ve en la f√≥rmula puede complicarse de tal forma que hace muy dif√≠cil su c√°lculo de manera anal√≠tica. Aqu√≠ es donde entran en juego los m√©todos num√©ricos como Markov Chain Monte Carlo (MCMC). Dentro de los MCMC, hay diversos m√©todos, pero aqu√≠ el que voy a tratar es el algoritmo Metropolis-Hastings.

Veamos un ejemplo concreto:

En primer lugar, simulemos algunos datos. En concreto, 50 valores de una distribuci√≥n de Poisson con media 5.

\(X \sim Poi(\lambda = 5)\)

\(p(x|\lambda) = \frac{e^{-\lambda}\lambda^{x}}{x!}\)

df <- rpois(n = 50, lambda = 5)
barplot(table(df), main = 'Data distribution', ylab = 'Count', space = 0.2)

El segundo paso es definir el modelo, que en este caso, como hemos generado los datos bajo una Poisson, la verosimilitud tambi√©n ser√° una Poisson con par√°metro \(\lambda\). Por √∫ltimo, tenemos que fijar una distribuci√≥n a priori para nuestro par√°metro \(\lambda\) de inter√©s. En este caso, nuestra distribuci√≥n a priori ser√° una Gamma con par√°metros de forma \(\alpha\) (shape) y \(\beta\) (rate) tal que su funci√≥n de densidad:

\(g(\lambda; \alpha, \beta) = \frac{\beta^{\alpha}\lambda^{\alpha-1}e^{-\lambda\beta}}{\Gamma (\alpha)} \propto \lambda^{\alpha-1}e^{-\lambda\beta}\)

Para analizar este modelo de forma anal√≠tica es conveniente que la distribuci√≥n a priori sea una Gamma, ya que la distribuci√≥n a priori Gamma para \(\lambda\) es conjugada (las distribuciones a priori y posteriori coinciden), es decir, que nuestra distribuci√≥n a posteriori tambi√©n ser√° una Gamma. Los par√°metros de esta distribuci√≥n Gamma a posteriori son:

\(\lambda|X \sim G(\alpha + \sum X_i , \beta + n)\)

Esta distribuci√≥n a posteriori se ha podido calcular de forma anal√≠tica porque las distribuciones a priori y posteriori son conjugadas, pero esto no siempre es el caso, de ah√≠ que se requiera de m√©todos num√©ricos que solucionen este problema.

Algoritmo Metropolis-Hastings

El funcionamiento del algoritmo es el siguiente:

  1. Comenzamos con un valor inicial aleatorio de \(\lambda\), en este caso, 3.
  2. Saltamos a otro punto cualquiera. En este caso, bajo una distribuci√≥n normal con el valor actual de lambda \((\lambda_a)\) como media y distribuci√≥n t√≠pica fijada a 1. Este ser√° nuestro nuevo lambda propuesto, \(\lambda_p\)
  3. Calculamos la verosimilitud para el valor actual y propuesto de \(\lambda\), tal que \(L(\lambda|X_1..X_n)=\prod_{i=1}^{N}\frac{e^{-\lambda}\lambda^{x_i}}{X_i!}\)
  4. Calculamos las probabilidades a posteriori como el producto entre la verosimilitud y la distribuci√≥n a priori. Despu√©s, calculamos el ratio de aceptaci√≥n, que se obtiene al dividir las probabilidades a posteriori del lambda propupuesto \((\lambda_p)\) y el lambda actual \((\lambda_a)\), consiguiendo as√≠ cancelar la evidencia, \(P(x)\), que es el t√©rmino del que precisamente quer√≠amos deshacernos:

\(\frac{P(\lambda_p | x)}{P(\lambda_a | x)}\frac{\frac{P(x|\lambda_p)P(\lambda_p)}{P(x)}}{\frac{P(x|\lambda_a)P(\lambda_a)}{P(x)}} = \frac{P(x|\lambda_p)P(\lambda_p)}{P(x|\lambda_a)P(\lambda_a)}\)

  1. Si este ratio es mayor a un valor aleatorio entre 0 y 1, aceptamos la propuesta y tomamos \(\lambda_a = \lambda_p\). En caso contrario, mantenemos el valor actual, \(\lambda_a\). La l√≥gica es la siguiente, a trav√©s del ratio de aceptaci√≥n lo que estamos consiguiendo es que estamos recorriendo con mayor frecuencia las regiones con mayor probabilidad a posteriori en detrimento de oras con menor probabilidad.
  2. Repetimos \(N\) veces.

En c√≥digo, ser√≠a as√≠:

mcmc <- 
  function(iters, lambda_ini = 3, shape_prior = 5, rate_prior = 2) {
    
    # Valores iniciales
    lambda_a <- lambda_ini
    posterior <- lambda_a  
    
    # Iteramos n veces
    for (i in seq_len(iters)) {
      
      # Elegimos un nuevo candidato a partir de una distirbuci√≥n normal 
      # con actual valor de lambda como media y una sd de 1
      lambda_p <- rnorm(n = 1, mean = lambda_a, sd = 1)
      
      # Calculamos la verosimilitud 
      likelihood_a <- prod(dpois(x = df, lambda_a))
      likelihood_p <- prod(dpois(x = df, lambda_p))
      
      # Calculamos la probabilidad a priori de los valores actuales y propuestos
      prior_a <- dgamma(x = lambda_a, shape = shape_prior, rate = rate_prior)
      prior_p <- dgamma(x = lambda_p, shape = shape_prior, rate = rate_prior)
      
      # Probabilidad a posteriori
      p_current <- likelihood_a * prior_a
      p_proposal <- likelihood_p * prior_p
      
      # Ratio de aceptacion
      p_accept <- p_proposal / p_current
  
      # Si el rario es mayor a un valor arbitrario entre 0 y 1, aceptamos propuesta
      accept <- runif(1, 0, 1) < p_accept
      if (accept) lambda_a <- lambda_p 
      
      posterior <- c(posterior, lambda_a)
    }
    
    posterior
  }
